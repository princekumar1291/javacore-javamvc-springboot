collection framework in java
what is collection-> it is group of objects which are represented as single unit.
what is framework-> it is set of classes and interfaces which are used to represent and manipulate the collection of objects.
why we use collection framework-> it is used to store and manipulate group of objects. (package-> java.util )


list of Interfaces in Collection framework in java (with definition and features):

1. Collection (interface)
   - Definition: Root interface of the collection hierarchy. It defines the basic operations for all collection types (except maps).
   - Features: Base for List, Set, and Queue; supports methods like add, remove, clear, size, iterator.

2. List (interface)
   - Definition: Ordered collection (sequence) that allows duplicate elements. Elements are indexed.
   - Features: Preserves insertion order, allows random access by index, permits duplicates, supports null elements.

3. Set (interface)
   - Definition: A collection that does not allow duplicate elements.
   - Features: No duplicate elements, does not guarantee order, at most one null element (in some implementations).

4. SortedSet (interface)
   - Definition: A Set that maintains its elements in ascending order.
   - Features: Elements automatically sorted, additional methods for range view.

5. NavigableSet (interface)
   - Definition: Extends SortedSet with navigation methods for closest matches.
   - Features: Methods for lower, floor, ceiling, higher; supports descending order view.

6. Queue (interface)
   - Definition: Used to hold multiple elements prior to processing, typically in FIFO order.
   - Features: Supports insertion, removal, and inspection; supports different queue types like priority queues, deques.

7. Deque (interface)
   - Definition: Double-ended queue allowing insertion and removal from both ends.
   - Features: Supports stack and queue operations; efficient manipulation at both ends.

8. Map (interface)
   - Definition: An object that maps keys to values; does not extend Collection interface.
   - Features: No duplicate keys, each key maps to at most one value, allows one null key (in some implementations) and multiple null values.

9. SortedMap (interface)
   - Definition: A Map that maintains keys in ascending order.
   - Features: Sorted keys, range view operations.

10. NavigableMap (interface)
    - Definition: Extends SortedMap with navigation methods for closest matches by key.
    - Features: Methods for lowerKey, floorKey, ceilingKey, higherKey; supports descending map view.

Important classes of Collection framework (with definition and features):

1. ArrayList (class)
   - Definition: Resizable array implementation of List interface.
   - Features: Fast random access, maintains insertion order, allows duplicates & nulls, slower insert/remove in middle.

2. LinkedList (class)
   - Definition: Doubly-linked list implementation of List and Deque interfaces.
   - Features: Fast at adding/removing elements at ends, slow random access, allows duplicates & nulls.

3. Vector (class)
   - Definition: Synchronized, resizable array implementation of List.
   - Features: Thread-safe, slightly slower due to synchronization, preserves insertion order.

4. Stack (class)
   - Definition: Subclass of Vector; implements stack (LIFO) operations.
   - Features: push, pop, peek methods; thread-safe.

5. HashSet (class)
   - Definition: Implements Set, backed by a hash table.
   - Features: No duplicate elements, no guaranteed order, allows one null element.

6. LinkedHashSet (class)
   - Definition: Hash table and linked list for Set; maintains insertion order.
   - Features: No duplicates, preserves insertion order, one null element allowed.

7. TreeSet (class)
   - Definition: Implements NavigableSet, backed by red-black tree.
   - Features: Sorted order, no duplicates, no nulls, higher retrieval cost.

8. PriorityQueue (class)
   - Definition: Queue implementation with elements ordered by priority.
   - Features: Not thread-safe, no nulls, allows duplicates, orders elements via Comparable or Comparator.

9. HashMap (class)
   - Definition: Implements Map, backed by a hash table.
   - Features: No duplicate keys, allows one null key and multiple null values, no order maintained.

10. LinkedHashMap (class)
    - Definition: HashMap with maintained insertion order.
    - Features: Predictable iteration order, one null key, multiple null values.

11. TreeMap (class)
    - Definition: Implements NavigableMap, keys sorted via natural ordering or Comparator.
    - Features: No null keys, sorted order, allows null values.

12. Hashtable (class)
    - Definition: Legacy, synchronized Map implementation.
    - Features: Thread-safe, no null keys or values, slower due to synchronization.

Collection Hierarchy in Java:
- Collection (interface)
    - List (interface)
        - ArrayList (class)
        - LinkedList (class)
        - Vector (class)
        - Stack (class)
    - Set (interface)
        - HashSet (class)
        - LinkedHashSet (class)
        - SortedSet (interface)
            - NavigableSet (interface)
                - TreeSet (class)
    - Queue (interface)
        - PriorityQueue (class)
        - Deque (interface)
            - ArrayDeque (class)
- Map (interface)
    - HashMap (class)
    - LinkedHashMap (class)
    - SortedMap (interface)
        - NavigableMap (interface)
            - TreeMap (class)
    - Hashtable (class)

what is thread in java
thread is a lightweight sub-process that can run concurrently with other threads.

multithreading in java
1. multithreading is a mechanism in which multiple threads are created to execute multiple tasks concurrently.

runnable,thread class in java
runnable interface is used to create a thread by implementing the run method.
thread class is used to create a thread by extending the thread class.

class Thread1 implements Runnable {
  public void run() {
    System.out.println("Thread is running");
  }
}

class Thread2 extends Thread {
  public void run() {
    System.out.println("Thread is running");
  }
}

public class thread {
  public static void main(String[] args) {
    Thread1 myThread = new Thread1();
    Thread2 myThread2 = new Thread2();
    Thread thread = new Thread(myThread);
    thread.start();
    Thread thread2 = new Thread(myThread2);
    thread2.start();
  }
}
